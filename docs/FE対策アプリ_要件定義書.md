# 基本情報技術者試験 対策アプリ「FE Master」要件定義書

> **ドキュメントバージョン**: v1.2（開発プロセス追加）
> **作成日**: 2026-02-21
> **対象**: Claude Code での Mac アプリ実装用インプット
> **想定利用者**: IT エンジニア経験のない QA エンジニア（基本情報技術者試験の受験予定者）

---

## 1. プロジェクト概要

### 1.1 背景と目的

基本情報技術者試験（FE）は IPA（情報処理推進機構）が実施する国家試験であり、IT エンジニアの登竜門とされている。しかし、既存の対策アプリは以下の問題を抱えている。

- 4 択問題を解くだけで、解説が専門用語だらけで理解できない
- 不正解時の「なぜ間違いなのか」が説明されない
- 自分のレベルに合った問題が出ない（簡単すぎる or 難しすぎる）
- 体系的な学習（レジュメ）と演習（問題集）が分離している

本アプリは、**IT エンジニアではない人間でも理解できる解説**を最大の特徴とし、レジュメ学習と適応型練習問題を一つのアプリに統合する。

### 1.2 コンセプト

**「IT の専門家じゃなくても、"なるほど！"と思える基本情報技術者試験対策アプリ」**

- すべての解説は「日常生活のたとえ話」から始まる
- 正解だけでなく、不正解の選択肢がなぜ間違いなのかも解説する
- ユーザーのレベルに合わせて問題難易度が自動調整される
- レジュメを読んで理解 → 問題で実践、の学習サイクルが回せる

### 1.3 対象ユーザー

- IT エンジニア経験がない（または浅い）社会人
- 基本情報技術者試験を初めて受験する人
- 既存アプリの解説が難しくて挫折した経験がある人

## 2. 技術スタック

### 2.1 推奨構成

| 項目 | 技術 | 選定理由 |
|------|------|----------|
| フレームワーク | **Electron** | Mac デスクトップアプリとして動作。Web 技術ベースで実装しやすい |
| フロントエンド | **React + TypeScript** | コンポーネント指向で UI 構築が効率的。型安全性も確保 |
| スタイリング | **Tailwind CSS** | ユーティリティベースで迅速な UI 構築が可能 |
| データ管理 | **ローカル JSON ファイル** | 問題・レジュメデータを静的に同梱。API 不要でオフライン動作 |
| ユーザーデータ保存 | **electron-store** または **SQLite** | 学習進捗・レベル・回答履歴をローカルに永続化 |
| ビルド | **electron-builder** | Mac 向け `.dmg` / `.app` の生成 |

### 2.2 設計方針

- **完全オフライン動作**: 外部 API への通信は一切行わない。すべてのコンテンツはアプリに同梱
- **従量課金なし**: Claude API 等のランタイム呼び出しは行わない
- **データドリブン**: 問題・レジュメは JSON ファイルで管理し、コンテンツの追加・修正が容易な構造にする

アプリは大きく **2 つのモジュール** と **共通機能** で構成される。

```
FE Master
├── 📖 レジュメモジュール（学習）
│   ├── 科目 A 対応レジュメ
│   │   ├── テクノロジ系
│   │   ├── マネジメント系

├── 📝 練習問題モジュール（演習）
│   ├── 科目 A 問題（700 問）

    ├── ブックマーク機能
    └── 設定
```

## 4. レジュメモジュール（学習機能）

### 4.1 概要

基本情報技術者試験のシラバス（Ver.9.0 準拠）に基づき、試験範囲を体系的にカバーするレジュメを提供する。すべての内容は「IT エンジニアではない人」が読んでも理解できるレベルで記述する。

### 4.2 コンテンツ構造

レジュメは以下の階層で構成される。

```
大分類（カテゴリ）
└── 中分類（チャプター）
    └── 小分類（セクション）
        ├── 概要説明（日常のたとえ話付き）
        ├── 詳細解説
        ├── 重要用語リスト
        ├── 図解（可能な場合）
        └── 関連する練習問題へのリンク
```

### 4.6 図解について

- 図解は SVG または HTML/CSS で描画するインライン図を想定
- 複雑な概念（ネットワーク構成、データベース正規化、アルゴリズムのフロー等）には図解を付ける
- 図解が技術的に困難な場合は、テキストベースの図（ASCII アート風）やテーブル形式で代替可
- 図解の有無はセクションごとに `diagram` フィールドで管理

### 5.4 レベルシステム（適応型難易度調整）

#### レベル定義

| レベル | 難易度 | 想定正答率 | 問題の特徴 |
|--------|--------|------------|------------|
| Lv.1 | 超入門 | 90%以上 | 用語の意味を問う基本問題。日常的な言葉で出題 |
| Lv.2 | 入門 | 85% | 用語の意味をやや詳しく問う。2択まで絞れる問題 |
| Lv.3 | 基礎 | 80% | 基本的な概念の理解を問う。教科書レベル |
| Lv.4 | 基礎応用 | 70% | 複数の概念を組み合わせた問題 |
| Lv.5 | 標準 | 60% | 試験本番の平均的な難易度。合格ラインの目安 |
| Lv.6 | 標準応用 | 55% | やや込み入った問題。正確な理解が必要 |
| Lv.7 | 応用 | 45% | 複数分野の横断的な知識が必要 |
| Lv.8 | 発展 | 35% | 深い理解と応用力が問われる |
| Lv.9 | 高難度 | 25% | 過去問でも正答率が低い問題レベル |
| Lv.10 | 最難関 | 15%以下 | 試験でも差がつく最高難度 |

#### レベル変動アルゴリズム

```
初期レベル: Lv.1（新規ユーザー）

レベルアップ条件:
  - 同一カテゴリ内で「直近3問が全て正解」した場合 → そのカテゴリのレベル +1
  - カウント対象は「そのカテゴリで出題された直近3問」（他カテゴリの問題は無関係）
  - 最大レベル: Lv.10

レベルダウン条件:
  - 同一カテゴリ内で「直近3問が全て不正解」した場合 → そのカテゴリのレベル -1
  - カウント対象は「そのカテゴリで出題された直近3問」
  - 最小レベル: Lv.1

カウンタのリセット:
  - レベルが変動した時点で、そのカテゴリの連続正解/不正解カウンタは 0 にリセットする
  - 正解→不正解が混在した場合もカウンタはリセット（連続性が途切れた時点で 0 に戻る）
  - セッション（アプリ終了→再起動）をまたいでもカウンタは保持される

特殊ルール:
  - カテゴリごとに独立したレベルを管理する
    （例: セキュリティ Lv.5、データベース Lv.3 のように分野別に管理）
  - 総合レベルは各カテゴリレベルの「試験配点比例の加重平均」で算出
    （テクノロジ系: 重み 0.55、マネジメント系: 重み 0.20、ストラテジ系: 重み 0.25）
  - ユーザーが任意のレベルに手動で変更することも可能（設定画面から即時反映）
  - レベル変動は即時反映（次の出題から新レベルの問題が出る）
```

#### 出題ロジック

```
1. ユーザーが選択したモード（科目A / 科目B / 全範囲）を取得
2. 各カテゴリのレベルに基づいて問題プールを決定
   - 現在のレベルの問題: 70%
   - 現在のレベル ±1 の問題: 30%（復習 + チャレンジ）
3. 未回答の問題を優先出題
4. 過去に間違えた問題は一定間隔で再出題（間隔反復学習）
   - 1回間違い: 3問後に再出題
   - 2回間違い: 10問後に再出題
   - 3回以上間違い: 「苦手問題」としてマーク、重点的に再出題
5. 同一カテゴリから連続5問以上出ないようにシャッフル
```

### 5.5 回答・解説フロー

```
[問題表示]
  ↓
[ユーザーが選択肢を選ぶ]
  ↓
[正誤判定 + アニメーション]
  ↓
[解説表示]
  ├── 正解の場合:
  │   ├── 「正解！」メッセージ
  │   ├── なぜその答えが正しいのかの解説（たとえ話付き）
  │   ├── さらに深掘り解説（展開式）
  │   └── 関連知識の補足
  │
  └── 不正解の場合:
      ├── 「残念！」メッセージ（ただし励ましのトーン）
      ├── 正解がどれかを表示
      ├── なぜ選んだ答えが間違いなのかの解説（たとえ話付き）
      ├── 正解の解説（たとえ話付き）
      ├── さらに深掘り解説（展開式）
      └── 関連するレジュメセクションへのリンク
  ↓
[次の問題へ / レジュメを読む / 終了する]
```

### 5.6 科目 B 問題の特殊仕様

- **実行イメージ**: 各ステップで何が起きているかを日常のたとえで解説

- **出題モード切替**: 科目 A のみ / 科目 B のみ / 全範囲

- **フォントサイズ調整**: 小/中/大
- **テーマ**: ライトモード / ダークモード

## 9. コンテンツ作成ガイドライン

### 9.1 解説の 5 原則

本アプリの解説はすべて以下の原則に従う。

1. **たとえ話ファースト**: まず日常生活のたとえ話で「こういうことだよ」と伝える
2. **段階的な深掘り**: 概要 → 詳細 → 関連知識、と段階的に情報を増やす
3. **不正解も解説する**: 4 つの選択肢すべてについて、なぜ正解/不正解なのかを説明する
4. **専門用語は必ず噛み砕く**: 初出の専門用語には必ず平易な説明を併記する
5. **ポジティブなトーン**: 間違えても落ち込まない。「ここがポイントだったんだ！」と前向きに

### 9.2 たとえ話の例

| IT 用語 | たとえ話 |
|---------|----------|
| CPU | 「会社でいうと社長さん。全体の指示を出して、計算もする一番忙しい人」 |
| メモリ（RAM） | 「作業机の広さ。机が広いほど一度にたくさんの資料を広げられる」 |
| ストレージ（HDD/SSD） | 「会社の倉庫。使わない資料はここに保管。必要になったら机に持ってくる」 |
| ファイアウォール | 「マンションの管理人さん。怪しい人は入れない、住人は自由に出入りできる」 |
| SQL | 「データベースへの指示書。『この条件に合うデータを持ってきて』とお願いする言語」 |
| 正規化 | 「散らかった部屋を整理整頓。同じものが2箇所にあったら片方にまとめる」 |
| プロトコル | 「通信のルールブック。お互いが同じルールを使わないと会話が成り立たない」 |
| 暗号化 | 「手紙を鍵付きの箱に入れて送ること。鍵を持っている人だけが読める」 |
| MTBF | 「機械が壊れるまでの平均時間。長いほど信頼性が高い」 |
| アジャイル開発 | 「料理を1品ずつ出すコース料理。少しずつ作って味見しながら進める方式」 |

### 9.3 科目 B 擬似言語の解説ルール

- 擬似言語コードの各行に「日本語で何をしているか」のコメントを付ける
- トレース表で変数の値の変化をステップごとに見せる
- 「この部分は、つまり○○をしている」とまとめを入れる
- アルゴリズム全体の「やりたいこと」を最初に日常のたとえで説明する

## 10. 問題作成時の品質基準

### 10.1 問題の品質チェックリスト

- [ ] 問題文は明確で曖昧さがないか
- [ ] 4 つの選択肢がすべて妥当な「ひっかけ」になっているか（明らかに変な選択肢がないか）
- [ ] 正解が 1 つだけであるか
- [ ] 各選択肢の解説が書かれているか（正解理由 + 不正解理由）
- [ ] たとえ話が含まれているか
- [ ] 難易度レベルが適切か
- [ ] カテゴリ・タグが正しく設定されているか
- [ ] 科目 B の場合、擬似言語コードとトレース表があるか

### 10.2 レベル別の問題作成指針

| レベル | 問題の特徴 | 解説の特徴 |
|--------|------------|------------|
| Lv.1-2 | 「○○とは何か」形式。選択肢の違いが明確 | たとえ話中心。専門用語は最小限 |
| Lv.3-4 | 基本的な知識の確認。やや紛らわしい選択肢あり | たとえ話 + 基本的な技術説明 |
| Lv.5-6 | 本番試験レベル。正確な理解が必要 | たとえ話 + 詳細な技術説明 + 関連知識 |
| Lv.7-8 | 複数概念の組み合わせ。応用力が必要 | 比較表やフロー図を使った解説 |
| Lv.9-10 | 高度な理解と判断力。紛らわしい選択肢多数 | 体系的な整理 + 試験テクニック |

## 12. 模擬試験モード

### 12.1 概要

本番の試験形式を模した模擬試験モードを提供する。時間制限付きで、本番に近い体験ができる。

### 12.2 モード仕様

| モード | 問題数 | 制限時間 | 出題範囲 |
|--------|--------|----------|----------|
| 科目 A 模擬試験 | 60 問 | 90 分 | テクノロジ 41問 + マネジメント 7問 + ストラテジ 12問 |
| 科目 B 模擬試験 | 20 問 | 100 分 | アルゴリズム 16問 + セキュリティ 4問 |
| ミニテスト（科目A） | 10 問 | 15 分 | ランダム |
| ミニテスト（科目B） | 5 問 | 25 分 | ランダム |

### 12.3 模擬試験の機能

- **タイマー表示**: 残り時間をヘッダーに常時表示
- **問題一覧パネル**: 回答済み/未回答/フラグ付きの問題をリスト表示
- **フラグ機能**: 後で見直したい問題にフラグを立てる
- **中断・再開**: 途中で中断し、後から再開可能
- **結果画面**: 終了後にスコア、カテゴリ別正答率、合格ライン判定を表示
- **解説モード**: 模擬試験終了後に全問の解説を通しで確認可能

### 12.4 スコア計算

- 単純正答率方式（IRT は実装が複雑なため、正答数 / 出題数 × 1000 点で算出）
- 600 点以上で「合格ライン到達」と表示
- カテゴリ別のスコア内訳も表示

## 14. 画面遷移フロー

### 14.1 メインフロー

```
ダッシュボード ─── レジュメ一覧 ─── レジュメ詳細
     │                                    │
     │                              [関連問題へ]
     │                                    ↓
     ├──── 練習問題 ─── 出題画面 ─── 解説画面 ─── 次の問題 / レジュメへ
     │         │
     │         ├── 科目選択（A / B / 全範囲）
     │         └── カテゴリ選択（任意）
     │
     ├──── 模擬試験 ─── 模擬試験中 ─── 結果画面 ─── 解説一覧
     │
     ├──── 学習進捗
     │
     ├──── ブックマーク一覧 ─── 問題詳細 / レジュメ詳細
     │
     ├──── 苦手問題一覧 ─── 問題詳細
     │
     └──── 設定
```

### 14.2 画面遷移の詳細ルール

- **レジュメ→問題**: レジュメ内の「関連問題リンク」をクリック → 練習問題の出題画面に遷移。戻るボタンでレジュメに復帰
- **解説→レジュメ**: 解説画面の「レジュメを読む」→ 関連するレジュメセクションに遷移。戻るボタンで解説画面に復帰
- **ブックマーク削除**: 一覧画面で各アイテムにゴミ箱アイコン。タップで確認なしに即削除（元に戻すは Cmd+Z）
- **設定変更**: レベル手動変更、テーマ変更等はすべて即時反映

## 15. エラーハンドリング

### 15.1 想定されるエラーと対応

| エラー | 対応 |
|--------|------|
| JSON データの読み込み失敗 | エラーメッセージ表示 + 該当カテゴリをスキップして他の機能は利用可能にする |
| ユーザーデータの破損 | バックアップから復元を試みる。不可能な場合は初期化を提案（確認ダイアログ付き） |
| ディスク容量不足で保存失敗 | 警告メッセージ表示 + 容量確保を促す |
| アプリ異常終了 | 次回起動時に直近のデータを自動復旧。定期自動保存（30秒間隔）で損失最小化 |

### 15.2 データ保全

- **自動保存**: 問題回答のたび + 30 秒間隔で自動保存
- **バックアップ**: ユーザーデータは直近 3 世代のバックアップを保持
- **バージョン互換**: アプリ更新時は既存ユーザーデータのマイグレーション処理を実行
